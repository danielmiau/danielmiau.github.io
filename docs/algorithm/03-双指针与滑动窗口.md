# LeetCode.239 滑动窗口最大值（Top100）

(https://leetcode.cn/problems/sliding-window-maximum/)

给你一个整数数组 `nums`，有一个大小为 `k` 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 `k` 个数字。滑动窗口每次只向右移动一位。

返回 *滑动窗口中的最大值* 。

**示例 1：**

```
输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
解释：
滑动窗口的位置                最大值
---------------               -----
[1  3  -1] -3  5  3  6  7       3
 1 [3  -1  -3] 5  3  6  7       3
 1  3 [-1  -3  5] 3  6  7       5
 1  3  -1 [-3  5  3] 6  7       5
 1  3  -1  -3 [5  3  6] 7       6
 1  3  -1  -3  5 [3  6  7]      7
```

**示例 2：**

```
输入：nums = [1], k = 1
输出：[1]
```

**提示：**

- `1 <= nums.length <= 105`
- `-104 <= nums[i] <= 104`
- `1 <= k <= nums.length`



**分析：** 使用普通暴力法每次窗口移动都要遍历一遍窗口找最大值，时间复杂度 O (nk)，会超时。这道题我们需要用 Deque 来维护一个单调递减队列，Deque 的核心作用是：队首永远是当前窗口的最大值。队首存下标，方便判断元素是否已经滑出窗口。

每次滑动窗口时，我们需要完成三件事：

1. 把队首超出窗口范围的元素踢出去
2. 把队尾所有比当前元素小的元素清掉，再把当前元素入队
3. 窗口每形成一次，就把队首的值加入结果数组

代码如下：

```java
public int[] maxSlidingWindow(int[] nums, int k) {
	// 判空
	if (nums == null || nums.length == 0 || k <= 0) return new int[0];
	// 获取数组长度
	int n = nums.length;
	// 获取结果集长度
    int count = n - (k - 1);
    // 创建结果集
    int[] res = new int[count];
    Deque<Integer> stack = new ArrayDeque<>();
    for (int i = 0; i < n; i++) {
        // 1. 队首维护：超出窗口范围的出队
        while (!stack.isEmpty() && stack.peekFirst() < i - k + 1) {
            stack.pollFirst();
        }
        // 2. 队尾维护：比当前元素小的全部出队
        while (!stack.isEmpty() && nums[stack.peekLast()] <= nums[i]) {
            stack.pollLast();
        }
        // 3. 加入当前下标
        stack.offerLast(i);
        // 3. 窗口形成后记录结果
        if (i >= k - 1) {
            res[i - k + 1] = nums[stack.peekFirst()];
        }
    }
    return res;
}
```

**步骤拆解**（关键顺序不能乱）

1. **清理队首**：把下标小于窗口左边界（`i - k + 1`）的元素移除。
2. **清理队尾**：把所有小于当前值的队尾元素移除（它们永远不可能成为窗口最大值）。
3. **加入当前下标**：维护队列的单调递减性。
4. **记录结果**：当窗口完全形成（`i >= k - 1`）时，队首元素就是当前窗口的最大值，加入结果数组。

**关键细节**

- 队列存**下标**，而非数值，方便判断是否越界。
- 必须先清理队首，再处理队尾，最后加入新元素。
- 窗口从 `i >= k - 1` 开始记录结果。

**总结：** 用 **双端队列（Deque）** 维护一个**单调递减队列**，队首始终是当前窗口的最大值。队列中存的是数组下标，方便判断元素是否滑出窗口。

# LeetCode.76 最小覆盖子串

(https://leetcode.cn/problems/minimum-window-substring/)

给定两个字符串 `s` 和 `t`，长度分别是 `m` 和 `n`，返回 s 中的 **最短窗口 子串**，使得该子串包含 `t` 中的每一个字符（**包括重复字符**）。如果没有这样的子串，返回空字符串 `""`。

测试用例保证答案唯一。

**示例 1：**

```
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。
```

**示例 2：**

```
输入：s = "a", t = "a"
输出："a"
解释：整个字符串 s 是最小覆盖子串。
```

**示例 3:**

```
输入: s = "a", t = "aa"
输出: ""
解释: t 中两个字符 'a' 均应包含在 s 的子串中，
因此没有符合条件的子字符串，返回空字符串。
```

**提示：**

- `m == s.length`
- `n == t.length`
- `1 <= m, n <= 105`
- `s` 和 `t` 由英文字母组成

**进阶：**你能设计一个在 `O(m + n)` 时间内解决此问题的算法吗？



**分析：**这道题是滑动窗口的一个经典场景，属于动态滑动窗口，即窗口的大小不固定，对于此类型的题我们通常使用一左一右两个指针，在长字符串上滑动，寻找符合短字符串要求的最小窗口，即：

**先扩张右指针**：把 `right` 不断往右移，直到窗口 `[left, right]` 包含了 `t` 中所有字符（包括重复次数）。

**再收缩左指针**：窗口满足条件后，尝试把 `left` 往右缩，同时记录当前窗口长度，如果依然满足条件，就更新最小窗口。

**重复扩张 + 收缩**：直到 `right` 遍历完整个 `s`。

**核心细节**（必须用「计数」来判断）

光靠 “有没有这个字符” 判断是不够的，因为 `t` 里可能有重复字符（比如 `t = "AABC"`），所以你需要两个关键数据结构：

- 一个哈希表 / 数组 `need`：记录 `t` 中每个字符需要的数量。
- 一个哈希表 / 数组 `window`：记录当前窗口中每个字符的数量。
- 一个变量 `valid`：记录窗口中满足 `need` 要求的字符种类数，当 `valid == need.size()` 时，窗口就是合法的。

**代码实现：**

```java
public String minWindow(String s, String t) {
	// 统计t中每个字符的需求数量
    Map<Character, Integer> need = new HashMap<>();
    Map<Character, Integer> window = new HashMap<>();
    for (char c : t.toCharArray()) {
        need.put(c, need.getOrDefault(c, 0) + 1);
    }

    int left = 0, right = 0;
    int valid = 0; // 窗口中满足need的字符种类数
    int start = 0, len = Integer.MAX_VALUE; // 记录最小窗口的起始位置和长度

    while (right < s.length()) {
        // 1. 扩张右指针，加入字符
        char c = s.charAt(right);
        right++;
        if (need.containsKey(c)) {
            window.put(c, window.getOrDefault(c, 0) + 1);
            if (window.get(c).equals(need.get(c))) {
                valid++;
            }
        }

        // 2. 窗口合法，收缩左指针
        while (valid == need.size()) {
            // 更新最小窗口
            if (right - left < len) {
                start = left;
                len = right - left;
            }

            char d = s.charAt(left);
            left++;
            if (need.containsKey(d)) {
                if (window.get(d).equals(need.get(d))) {
                    valid--;
                }
                window.put(d, window.get(d) - 1);
            }
        }
    }

    return len == Integer.MAX_VALUE ? "" : s.substring(start, start + len);
}
```

**完整步骤**

1. 初始化：
   - 统计 `t` 中所有字符的出现次数，存入 `need`。
   - 初始化 `left = 0, right = 0, valid = 0, start = 0, len = ∞`。
2. 扩张右指针：
   - 遍历 `s`，移动 `right`，把 `s[right]` 加入 `window`。
   - 如果 `window[s[right]] == need[s[right]]`，说明该字符已满足要求，`valid++`。
3. 窗口合法时收缩左指针：
   - 当 valid == need.size() 时，尝试移动 left：
     - 如果当前窗口长度小于 `len`，更新 `len` 和 `start`。
     - 把 `s[left]` 从 `window` 中减去。
     - 如果 `window[s[left]] < need[s[left]]`，说明该字符不再满足要求，`valid--`。
     - `left++`。
4. 返回结果：
   - 如果 `len` 还是无穷大，说明没有合法窗口，返回 `""`；否则返回 `s.substring(start, start + len)`。
