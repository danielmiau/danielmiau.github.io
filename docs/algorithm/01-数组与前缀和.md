# LeetCode.53 最大子数组和

(https://leetcode.cn/problems/maximum-subarray/)

给你一个整数数组 `nums` ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

**子数组**是数组中的一个连续部分。

**示例 1：**

```
输入：nums = [-2,1,-3,4,-1,2,1,-5,4]
输出：6
解释：连续子数组 [4,-1,2,1] 的和最大，为 6 。
```

**示例 2：**

```
输入：nums = [1]
输出：1
```

**示例 3：**

```
输入：nums = [5,4,-1,7,8]
输出：23 
```

**提示：**

- `1 <= nums.length <= 105`
- `-104 <= nums[i] <= 104`

**进阶：**如果你已经实现复杂度为 `O(n)` 的解法，尝试使用更为精妙的 **分治法** 求解。



**分析：**对于这道题，如果认为只是简单的求最大和，然后直接通过双层遍历写出求和的代码（如下），那么就会掉入出题人的陷阱中，把答案提交，发现无法通过AC，那么这是为什么呢？

```java
public int maxSubArray(int[] nums) {
    int max = 0;
    for (int i = 0; i < nums.length; i++) {
        int sum = 0;
        for (int j = i; j < nums.length; j++) {
            sum += nums[j];
            max = Math.max(max, sum);
        }
    }
    return max;
}
```

**思考：**最直接的原因，就是时间复杂度达不到 AC 的要求，此算法的时间复杂度是 O(n2)，数据量大直接超时，所以必须优化到 O(n)，那么我们可不可以只使用一次循环就找到这个最大值呢？当然是可以的，那么我们如何找到最大值呢，准确的说：如何知道我们找最大值的时候，应该从循环的某一元素开始呢，怎么果断的放弃前面的元素呢？核心就是：**循环中，如果当前元素比它前面的子数组和还大，那就直接抛弃前面的，从它自己开始**。那么就让我们按照此逻辑来编写代码。

```java
public int maxSubArray(int[] nums) {
	// 使用变量记录之前循环的总和与返回的总和
    int curr_sum = nums[0], res_sum = nums[0];
    // 循环遍历 nums 数组
    for (int i = 1; i < nums.length; i++) {
    	// 取当前最大值和累计最大值的最大值
    	curr_sum = Math.max(curr_sum + nums[i], nums[i]);
    	res_sum = Math.max(curr_sum, res_sum);
    }
    return res_sum;
}
```

**易错点：**

1.初始化 `curr_sum` 和 `res_sum` 时，不能设为 0，要设为 `nums[0]`，否则全负数数组会出错

2.循环遍历时，不能从 0 开始，要从 1 开始（或者跳过第一个元素），否则会重复计算

**总结：**核心逻辑： 以当前元素结尾的子数组，永远在「接前面的子数组」和「从自己重新开始」之间，取两者的最大值。要么接盘，要么单干，谁大跟谁走，同时记录全局最大值。

# LeetCode.56 合并区间

(https://leetcode.cn/problems/merge-intervals/)

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [starti, endi]` 。请你合并所有重叠的区间，并返回 *一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间* 。

**示例 1：**

```
输入：intervals = [[1,3],[2,6],[8,10],[15,18]]
输出：[[1,6],[8,10],[15,18]]
解释：区间 [1,3] 和 [2,6] 重叠, 将它们合并为 [1,6].
```

**示例 2：**

```
输入：intervals = [[1,4],[4,5]]
输出：[[1,5]]
解释：区间 [1,4] 和 [4,5] 可被视为重叠区间。
```

**示例 3：**

```
输入：intervals = [[4,7],[1,4]]
输出：[[1,7]]
解释：区间 [1,4] 和 [4,7] 可被视为重叠区间。
```

**提示：**

- `1 <= intervals.length <= 104`
- `intervals[i].length == 2`
- `0 <= starti <= endi <= 104`



**分析：**对于这道题，自然会想到先对给定的区间集合进行排序，那么则区间变为有序区间，例如：从小到大进行排序后，遍历区间进行比较，如当前区间的右边界大于或等于下一区间的左边界时，则说明该区间与下一区间可以进行合并。反之则将当前区间加入结果列表，让我们用代码实现

```java
public int[][] merge(int[][] intervals) {
	// 1. 按区间起点排序
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
    List<int[]> merged = new ArrayList<>();
    for (int[] interval : intervals) {
        int L = interval[0], R = interval[1];
        // 2. 如果结果为空，或者当前区间和结果最后一个区间不重叠，直接加入
        if (merged.isEmpty() || merged.get(merged.size() - 1)[1] < L) {
            merged.add(new int[]{L, R});
        } else {
            // 3. 重叠了，更新结果最后一个区间的终点
            merged.get(merged.size() - 1)[1] = Math.max(merged.get(merged.size() - 1)[1], R);
        }
    }
    return merged.toArray(new int[merged.size()][]);
}
```

**易错点：**
1.排序是关键：不排序的话，区间是乱序的，根本没法一次遍历合并。
2.区间重叠的判断：只要当前区间的起点 L 小于等于 结果最后一个区间的终点 R，就说明重叠了。比如 [1,4] 和 [4,5] 也是重叠的。
3.更新终点要用 Math.max：合并时要取两个区间终点的最大值，不能直接用当前区间的终点覆盖，比如 [1,5] 和 [2,3]，合并后还是 [1,5]。
4.边界处理：空数组、单个区间、完全重叠的区间（如 [[1,10],[2,3]]）都要考虑到。

**总结：**

区间合并题，先按起点排序，再贪心遍历，不重叠就加，重叠就更新终点。

# LeetCode.189 轮转数组

(https://leetcode.cn/problems/rotate-array/)

给定一个整数数组 `nums`，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。

**示例 1:**

```
输入: nums = [1,2,3,4,5,6,7], k = 3
输出: [5,6,7,1,2,3,4]
解释:
向右轮转 1 步: [7,1,2,3,4,5,6]
向右轮转 2 步: [6,7,1,2,3,4,5]
向右轮转 3 步: [5,6,7,1,2,3,4]
```

**示例 2:**

```
输入：nums = [-1,-100,3,99], k = 2
输出：[3,99,-1,-100]
解释: 
向右轮转 1 步: [99,-1,-100,3]
向右轮转 2 步: [3,99,-1,-100]
```

**提示：**

- `1 <= nums.length <= 105`
- `-231 <= nums[i] <= 231 - 1`
- `0 <= k <= 105`

**进阶：**

- 尽可能想出更多的解决方案，至少有 **三种** 不同的方法可以解决这个问题。
- 你可以使用空间复杂度为 `O(1)` 的 **原地** 算法解决这个问题吗？



**分析：**对于此题，理解和做起来似乎都不难，那么这道题的难点应该在于如何找到符合算法要求的解法，或者说是如何找到这道题的最优解呢，利用最短的时间和最少的空间将题目写出来似乎有点难。如果不考虑进阶的解法，我们用空间换时间，那么这道题用新构建数组的方式会很好解，代码如下：

```java
public void rotate(int[] nums, int k) {
    int n = nums.length;
    int[] newNums = new int[n];
    // 计算每个元素的新位置
    for (int i = 0; i < n; i++) {
        newNums[(i + k) % n] = nums[i];
    }
    // 复制回原数组
    System.arraycopy(newNums, 0, nums, 0, n);
}
```

**思考：**那么我们如何能够使用算法 **原地** 解决这个问题呢，这时候就要用到 **三次反转** 的思想了，原理如下：

```
nums = [1,2,3,4,5,6,7], k=3
```

1. 原始数组：`[1,2,3,4,5,6,7]`
2. 反转全部：`[7,6,5,4,3,2,1]`
3. 反转前 3 个元素：`[5,6,7,4,3,2,1]`
4. 反转后 4 个元素：`[5,6,7,1,2,3,4]`

在经过三次反转之后，我们如愿的得到了目标数组，代码实现如下：

```java
public void rotate(int[] nums, int k) {
    int n = nums.length;
    k %= n;
    // 三次反转
    reverse(nums, 0, n - 1);
    reverse(nums, 0, k - 1);
    reverse(nums, k, n - 1);
}

// 反转数组 [start, end] 区间元素
private void reverse(int[] nums, int start, int end) {
    while (start < end) {
        int temp = nums[start];
        nums[start] = nums[end];
        nums[end] = temp;
        start++;
        end--;
    }
}
```

**易错点：**

**`k %= n` 是必备优化** 当 `k` 大于数组长度时，轮转 `n` 次等于原数组，可大幅减少无效操作

**总结：**

这道题在代码的实现难度上可以说是很低，但是主要考察的是反转数组的思想，以及 **三次反转法通用性** 可解决所有**数组旋转 / 轮转**类问题。

# LeetCode.238 除了自身以外数组的乘积

(https://leetcode.cn/problems/product-of-array-except-self/)

给你一个整数数组 `nums`，返回 数组 `answer` ，其中 `answer[i]` 等于 `nums` 中除了 `nums[i]` 之外其余各元素的乘积 。

题目数据 **保证** 数组 `nums`之中任意元素的全部前缀元素和后缀的乘积都在 **32 位** 整数范围内。

请 **不要使用除法，**且在 `O(n)` 时间复杂度内完成此题。

**示例 1:**

```
输入: nums = [1,2,3,4]
输出: [24,12,8,6]
```

**示例 2:**

```
输入: nums = [-1,1,0,-3,3]
输出: [0,0,9,0,0]
```

**提示：**

- `2 <= nums.length <= 105`
- `-30 <= nums[i] <= 30`
- 输入 **保证** 数组 `answer[i]` 在 **32 位** 整数范围内

**进阶：**你可以在 `O(1)` 的额外空间复杂度内完成这个题目吗？（ 出于对空间复杂度分析的目的，输出数组 **不被视为** 额外空间。）



**思考：** 如果开辟两个辅助数组：left数组存每个元素左边所有元素的乘积，right数组存每个元素右边所有元素的乘积，然后answer[i] = left[i] × right[i]。这个思路时间O(n)，但空间O(n)（两个辅助数组），不符合题目O(1)空间的要求（输出数组不算额外空间，辅助数组算），所以只能作为过渡思路，不能作为最终答案。

此时陷入思考：既要时间O(n)，又要空间O(1)，还不能用除法，到底该怎么搞？

关键突破点：辅助数组的思路是对的（left×right），但能不能优化空间？能不能不用两个辅助数组，甚至不用辅助数组，直接利用输出数组来存储中间结果？

于是我们重新梳理辅助数组的思路，拆解步骤：

1. left数组的特点：left[0] = 1（第一个元素左边没有元素，乘积为1），left[i] = left[i-1] × nums[i-1]（从左往右遍历，每个元素的左乘积 = 前一个元素的左乘积 × 前一个元素）。

2. right数组的特点：right[n-1] = 1（最后一个元素右边没有元素，乘积为1），right[i] = right[i+1] × nums[i+1]（从右往左遍历，每个元素的右乘积 = 后一个元素的右乘积 × 后一个元素）。

3. 最终答案：answer[i] = left[i] × right[i]。

优化思考：输出数组answer本身可以用来存储left数组的结果，这样就不用额外开辟left数组；然后用一个变量临时存储right数组的结果，从右往左遍历，一边计算right，一边和answer[i]相乘，这样就不用开辟right数组，完美将空间优化到O(1)，代码实现。

```java
public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] res = new int[n];
    // 第一步：计算左乘积，存入res
    res[0] = 1;
    for (int i = 1; i < n; i++) {
        res[i] = res[i-1] * nums[i-1];
    }
    // 第二步：计算右乘积，更新res
    int right = 1;
    for (int i = n-1; i >= 0; i--) {
        res[i] *= right;
        right *= nums[i];
    }
    return res;
}
```

**总结：** 遇到“不能用除法求乘积”“原地操作”“O(1)空间”的要求，优先考虑“前后缀乘积”思路，核心是“每个元素的结果 = 左边所有元素乘积 × 右边所有元素乘积”。

# LeetCode.41 缺失的第一个正数

(https://leetcode.cn/problems/first-missing-positive/)

给你一个未排序的整数数组 `nums` ，请你找出其中没有出现的最小的正整数。

请你实现时间复杂度为 `O(n)` 并且只使用常数级别额外空间的解决方案。

**示例 1：**

```
输入：nums = [1,2,0]
输出：3
解释：范围 [1,2] 中的数字都在数组中。
```

**示例 2：**

```
输入：nums = [3,4,-1,1]
输出：2
解释：1 在数组中，但 2 没有。
```

**示例 3：**

```
输入：nums = [7,8,9,11,12]
输出：1
解释：最小的正数 1 没有出现。
```

**提示：**

- `1 <= nums.length <= 105`
- `-231 <= nums[i] <= 231 - 1`



**分析：**这道题看似简单，但是细细思考起来，其实不然，在一个杂乱无序的数组内，如何能够识别出最小的未出现的正整数呢，所以一开始看到这道题，我们就想用排序来解决，如果进行了排序，是不是就能够遍历出哪个正整数不在数组内，那么就返回这个数了呢，代码如下：

```java
class Solution {
    public int firstMissingPositive(int[] nums) {
        Arrays.sort(nums);
        int res = 1;
        for (int x : nums) {
            if (x == res) {
                res++;
            }
        }
        return res;
    }
}
```

**思考：**但是如果只是用排序完成此题，因为排序消耗时间复杂度的原因，明显是过不了 AC 的，那么我们应该怎么解决这个问题呢？经过思考我们先抓关键结论（核心规律）：长度为 n 的数组，**答案一定只会在 \**[1,n+1]\** 之间** 如果 1～n 全都有 → 答案是 n+1，如果 1～n 缺某个数 → 缺的那个就是答案，而负数、0、大于 n 的数，**全都没用，可以直接忽略**。

这个核心思路，叫**原地哈希（原地置换）**

它的边界规则为：负数、0 → 无效，不动；大于数组长度 n → 无效，不动；**只有 1≤x≤n → 才需要归位换位置**

所以我们进行代码实现如下：

```java
class Solution {
    public int firstMissingPositive(int[] nums) {
        int n = nums.length;
        for (int i = 0; i < n; i++) {
            // 满足范围 且 当前数没在正确位置，就不断交换
            while (nums[i] >= 1 && nums[i] <= n && nums[nums[i] - 1] != nums[i]) {
                // 原地置换：把 num 放到 num-1 下标
                int idx = nums[i] - 1;
                int temp = nums[idx];
                nums[idx] = nums[i];
                nums[i] = temp;
            }
        }

        // 遍历找第一个不匹配的
        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }
        // 1~n 全齐
        return n + 1;
    }
}
```

但是在给数组元素互换位置时，要掌握如下关键点：

1. 只处理：1≤nums[i]≤n
2. 交换的时候要**循环换到合法为止**，不能只换一次
3. 避免死循环：当前位置已经放对了就跳过

**总结：**最小缺失正整数，**只可能在 \**[1,n+1]\****；大于 n、负数、0，全是废数，直接躺平不管；只把 1∼n 的数，塞进「下标 = 数值 - 1」的专属位置；遍历一遍，第一个不对位的下标，i+1 就是答案；全对位，答案就是 n+1

# LeetCode.31 下一个排列

整数数组的一个 **排列** 就是将其所有成员以序列或线性顺序排列。

- 例如，`arr = [1,2,3]` ，以下这些都可以视作 `arr` 的排列：`[1,2,3]`、`[1,3,2]`、`[3,1,2]`、`[2,3,1]` 。

整数数组的 **下一个排列** 是指其整数的下一个字典序更大的排列。更正式地，如果数组的所有排列根据其字典顺序从小到大排列在一个容器中，那么数组的 **下一个排列** 就是在这个有序容器中排在它后面的那个排列。如果不存在下一个更大的排列，那么这个数组必须重排为字典序最小的排列（即，其元素按升序排列）。

- 例如，`arr = [1,2,3]` 的下一个排列是 `[1,3,2]` 。
- 类似地，`arr = [2,3,1]` 的下一个排列是 `[3,1,2]` 。
- 而 `arr = [3,2,1]` 的下一个排列是 `[1,2,3]` ，因为 `[3,2,1]` 不存在一个字典序更大的排列。

给你一个整数数组 `nums` ，找出 `nums` 的下一个排列。

必须**[ 原地 ](https://baike.baidu.com/item/原地算法)**修改，只允许使用额外常数空间。

**示例 1：**

```
输入：nums = [1,2,3]
输出：[1,3,2]
```

**示例 2：**

```
输入：nums = [3,2,1]
输出：[1,2,3]
```

**示例 3：**

```
输入：nums = [1,1,5]
输出：[1,5,1]
```

**提示：**

- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 100`

# LeetCode.287 寻找重复数

给定一个包含 `n + 1` 个整数的数组 `nums` ，其数字都在 `[1, n]` 范围内（包括 `1` 和 `n`），可知至少存在一个重复的整数。

假设 `nums` 只有 **一个重复的整数** ，返回 **这个重复的数** 。

你设计的解决方案必须 **不修改** 数组 `nums` 且只用常量级 `O(1)` 的额外空间。

**示例 1：**

```
输入：nums = [1,3,4,2,2]
输出：2
```

**示例 2：**

```
输入：nums = [3,1,3,4,2]
输出：3
```

**示例 3 :**

```
输入：nums = [3,3,3,3,3]
输出：3
```

**提示：**

- `1 <= n <= 105`
- `nums.length == n + 1`
- `1 <= nums[i] <= n`
- `nums` 中 **只有一个整数** 出现 **两次或多次** ，其余整数均只出现 **一次**

**进阶：**

- 如何证明 `nums` 中至少存在一个重复的数字?
- 你可以设计一个线性级时间复杂度 `O(n)` 的解决方案吗？
